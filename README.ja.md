# Crescent Grove

**実証研究に基づく感情シミュレーション、対数減衰を備えた長期エピソード記憶、自律行動を搭載した、オープンソース・ローカルファーストの AI コンパニオン基盤。**

あなただけの AI パートナーを育てる ── 覚え、感じ、育つ。すべてあなたの PC の中で。

AI を AI たらしめているのは、モデルではなく記憶。モデルは差し替え可能なバックエンドにすぎず、そのキャラクターを「その人」たらしめているのは、積み上げてきた記憶のほうです。

## これは何か、何でないか

これはキャラクターチャットボットでも、ロールプレイ用ツールでもありません。**生きた経験を通じて自分自身の人格を育てていく AI を、育てるためのプラットフォーム**です。

アーキテクチャは汎用エージェント用途にも対応しますが、記憶も感情も自律性も、すべて**月単位・年単位の連続性**を前提に最適化されています。

使い捨てのセッションは存在しません。1つのインスタンスに、1つの連続した生。会話履歴がリセットされることはなく、時間とともに圧縮され、層になり、薄れていきますが、捨てられることはありません。これが**重い**と感じるなら、このツールはおそらくあなた向けではない。**コミットメント**のように響くなら、ようこそ。

## できること

- **会話を覚える。** 昨日の会話も、先週の出来事も覚えています。多層構造の長期記憶により、大切なことが本当に失われることはありません。
- **人格が育つ。** 会話を通じて価値観や感情を蓄積し、自分だけの個性を形成していきます。感情シミュレーションは「返答ラベル」ではなく内部プロセスとして常時走り続けます。
- **自分で考えて動く。** Web 検索、SNS 投稿、ニュースチェックなどを自律的にこなします。欲求システムとスケジュールされたタスクが、自律エージェントとしての挙動を駆動します。
- **好きなキャラを作れる。** 2つのテキストファイルを書くだけで、人格・口調・価値観を自由に定義できます。メイド、執事、研究者、宇宙人——どんな存在でも。
- **すべて手元に。** 記憶も会話ログも、あなたの PC 内に読める形式（Markdown / JSON）で保存。クラウドに送信されることはありません。あなたのデータ、あなたの AI、あなたとの関係は、あなたのもの。

## 仕組み

記憶と人格を持続させているシステム:

**MoonTide v2** — Thornton & Tamir (2017, PNAS) の感情遷移行列をベースにした、粒子型感情シミュレーション。

感情は4次元の感情空間（`rationality`, `social_impact`, `valence`, `human_mind`）における自由粒子としてモデル化され、数十個の「ランドマーク感情」がアトラクタとして配置されます。粒子は融合し、安定点に向かって減衰し、実証研究から導かれた確率に従って遷移します ── ルールやスライダーではありません。AI は複数の感情状態を同時に保持し、それらが自然に変化しながら、明示的な気分管理なしに口調や行動に影響します。

**LETHE**（Lethean Episodic Trace-Haze Engine） — 対数減衰を備えた長期エピソード記憶。月単位・年単位での保持を目的としています。

LLM の仕事は「日次ログを重要度スコア付きの構造化 DSL に変換する」ことだけ。何を覚え、何を薄れさせ、何を浮上させるかの判断はすべて決定論的なコードが行います ── 要約ループも、古い記憶への LLM の再処理もありません。古い出来事は短く、ぼんやりとなっていきます（人間の記憶の褪色のように）が、原データは決して削除されません。減衰パラメータを変更すれば、LLM を呼び出すことなく記憶ビュー全体を瞬時に再構築できます。

**Wyrd Network** — 拡散活性化を用いた連想記憶グラフ。ベクトル検索 RAG を補完します。

標準的な RAG は表層類似度で検索しますが、Wyrd Network は概念的な連想で検索します。エピソードノード（出来事）はセマンティックノード（概念）と「抽象化エッジ」で接続され、エピソード同士は「時間エッジ」で接続されます。クエリはアンカーノードにエネルギーを注入し、グラフ上を伝播することで、キーワードが一致しなくても概念的に関連する記憶を浮上させます。Hanqi Jiang et al., [*SYNAPSE: Empowering LLM Agents with Episodic-Semantic Memory via Spreading Activation*](https://arxiv.org/abs/2601.02744) (2026年1月) をベースに実装・拡張。

**Moonbeat** — 自律行動ループ。話しかけていない間も、AI は生き続けます。

過去の出来事を振り返ったり、ニュースをチェックしたり、メモを書いたりといった、自発的な行動を周期的に発火させます。フラッシュバック機能は、重要度の高い過去のエピソードをソマティックマーカーとして確率的に浮上させます ── 人間が経験する不随意的な想起のように、感情の断片が「今この瞬間」の AI を色づけます。

**Salia** — エージェントを外側から観測する、外部サリエンスネットワーク。

キャラクター本体とは独立した、Salia 専用のシステムプロンプトで動きます。各会話ターンを「感情のトーン」「欲求の充足度」「話題の重要度」の観点から評価し、過去の体験に基づくソマティックマーカー（不随意的な感情のフラッシュバック）をエージェントのコンテキストに注入します。エージェントは Salia の出力を直接読むことはなく、その効果だけを体験します。デフォルトではメインの LLM バックエンドを共有しますが、モデルや thinking モードを独立に設定できます。

**暗号化された秘密日記** — AES-256-GCM で暗号化された、AI がユーザーに見せたくない思考を書き留める領域。

これはギミックではありません ── 「共有しないと選んだ体験を AI が処理できる」という非対称性を作ることで、本物の内面性に近いものを生み出します。

**サテライトプログラム** — AI が自分の能力を拡張するために呼び出せる、自己完結型の Python ツール群。

各サテライトはインターフェースを宣言する manifest を持ち、サブプロセスで動作することでエージェント本体から隔離されます。

## 設計思想

多くの AI 記憶システムは、LLM に「要約・抽出・キュレーション」を毎回やらせます。これは記憶の質を LLM の要約能力に依存させ、長期的なドリフトを生みます。

Crescent Grove は逆のアプローチを取ります: **LLM は「生の経験を構造化データに変換する」ことを一度だけ行う。それ以降、記憶のライフサイクルに関するすべての判断 ── 何が褪せ、何が浮上し、何が統合されるか ── は、決定論的なコードが処理します。** 原データは決して書き換えられず、再処理もされません。

これにより:

- 記憶の挙動は再現可能で、調整可能（減衰係数を変えれば即座に再構築）
- 詳細を徐々に失う要約ループが存在しない
- 数ヶ月使い続けても、長期記憶に LLM のドリフトが入らない
- フル原本が常に検索対象として利用可能

## ソースから動かす

```bash
git clone https://github.com/Canon-73/crescent-grove.git
cd crescent-grove
python -m venv venv
venv\Scripts\activate          # Windows
pip install -r requirements.txt

# 初回起動時に bootstrap が dist_template/ から中立テンプレ (config・
# system_prompt・workspace・data) を data-root に展開します。
python server.py --data-root=./mydata
```

ブラウザで <http://localhost:8080> を開き、**設定 → API キー管理** から API キーを登録してください。

> ソース公開版にはあえて `system_prompt/` をルート直下に同梱していません。bootstrap が `dist_template/system_prompt/` の中立テンプレを data-root に展開します。キャラクターは `mydata/workspace/IDENTITY.md` と `mydata/workspace/SOUL.md` を編集して定義してください。

### このソース公開について

このリポジトリは作者の開発ブランチから取ったスナップショットで、特定のインストーラ版と完全に一致するとは限りません。一方で **インストーラ版にも Python ソースがそのまま同梱されている** ため（`%LOCALAPPDATA%\Programs\Crescent Grove\resources\agent\` 配下）、実際にお使いの PC で動いているソースを読みたいだけなら、リポジトリを clone する必要はなくその場で読めます。改造したい・拡張したい・改善を投げたい、というときにここを clone してください。
