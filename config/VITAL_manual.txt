# VITAL/MENTAL 仕組みまとめ

## Stamina（体力）

**何を表すか**: 今日どれだけAPIを使ったかの疲労度。

**初期値**: 100（毎日リセット）

**減り方**: 会話するたびに、API消費額（USD）の増分に応じて減ります。
日次予算（デフォルト $0.50）を使い切ると 0 になる計算です。
※消費差分を round() でスタミナ消費量に変換して減算します。

**回復**: 1時間アクセスなしにつき +20 回復。
上限は100です。

**日付変更時**: 0時に stamina が 100 にリセットされます。

**閾値の効果**:
`data/vital_messages.json` の設定に従い、システムプロンプトに追加指示が注入されます。
（デフォルト: 30以下で体調メモ、0で睡眠モード）

---

## Mental（精神力）

**何を表すか**: 感情的なエネルギー。

**初期値**: 100（日付変更で**リセットされません**。日をまたいで持続します）

**減り方と回復**: 応答の最後に出力される `[VITAL_REPORT]` タグをバックエンドがパースすることで変動します。

| 変動タグ (例) | 変動量 | 理由の例 |
|---|---|---|
| `[VITAL_REPORT: mental -15, reason: ...]` | -15 | 嫌いな話題に触れて黙った |
| `[VITAL_REPORT: mental +20, reason: ...]` | +20 | 自分の意見を表明できた |

**自然回復**: 1時間アクセスなしにつき +5 回復。
上限は100です。

**閾値の効果**:
`data/vital_messages.json` の設定に従い、システムプロンプトに追加指示が注入されます。

---

## 全体のサイクル

1. **アクセス受信**: 前回のアクセスからの経過時間に基づき、stamina/mental が自然回復。
2. **APIコスト確認**: tracker が前回の呼び出し時からの消費差分を計算し、stamina を減算。
3. **プロンプト注入**: `vital_messages.json` から現在の値に合致するメッセージを読み込み、システムプロンプトの最後に結合。
4. **LLM応答**: 応答に含まれる `[VITAL_REPORT]` をパースして mental を更新し、タグをユーザーから隠蔽。
5. **保存**: 全ての数値を `data/vital.json` にアトミックに保存。

---

## 設定ファイル

### data/vital.json
現在の stamina, mental、および前回のAPI使用率（previous_ratio）を保持します。

### data/vital_messages.json
メッセージと閾値の定義。昇順に評価され、最初に条件を満たしたものが採用されます。
```json
{
  "stamina": [
    { "threshold": 0, "message": "睡眠プロンプト..." },
    { "threshold": 30, "message": "体調メモ..." }
  ],
  "mental": [
    { "threshold": 30, "message": "気持ちメモ..." }
  ]
}
```
